Isolation Forests Anomaly Detection¶

Isolation Forests do not suffer from the curse of dimensionality so we used it to detect anomalies on our subsetted data, which still has a large number of features. First we identified a set of outliers based every single feature in the data. The plots illustrate the positions of the outliers in 2-D and 3-D PCA space.

pred=lf.fit_predict(data)

We also ran an Isolation Forest on each feature individually. The first 2-D and 3-D plots show the number of anomalous features in the PCA space. The histogram and line plot illustrate how many outlier features the samples have. This is useful because it shows to what degree each sample is an outlier using this metric. We arbitrarily chose an outlier cutoff of 100 or more anomalous features (about a quarter of the features). The second set of 2-D and 3-D plots show the points that are above this cutoff, and therefore considered anomalies.

Local Outlier Factor Anomaly Detection¶

Local Outlier Factor (LOF) anomaly detection uses density to find outliers. Not knowing which combination of LOF metric, K nearest neighbor value, and algrithem would work best, we used every combination of the sklearn options, as well as a range of KNN values. Some combinations were incompatible and therefore could not be used. We used the results of each to vote on which samples would be considered a LOF outlier. The 2-D and 3-D plots below show the results.

In [10]:

from sklearn.neighbors import LocalOutlierFactor
temp = [0]*len(data)
LOFvoting = pd.DataFrame({"temp": temp})
for h in list(range(5,30,5)):
    for i in ["cityblock", "cosine", "euclidean", "l1", "l2", "manhattan", "minkowski"]:
        for j in ["auto", "ball_tree", "kd_tree", "brute"]:
            try:
                LOF = LocalOutlierFactor(algorithm = j, metric = i, n_neighbors = h)
                key = str(h) + i + j
                LOFvoting[key] = LOF.fit_predict(data)
            except:
                print("Error:", str(h), i, j)

One-Class Support Vector Machine Anomaly Detection¶

Anomaly Detection and Elimination¶

Rather than using one method of anomaly detection we decided to employ multiple methods and use the results of each to vote on which samples were outliers and which were not. We did this because our dataset is small and we wanted to rigorously test to make sure a sample was an outlier before eliminating it. This also helped us to minimize the effect of each model's short comings with respect to our data (like suffering from the curse of dimensionality, ect.).
